## 配置顺序依赖

```
步骤① 信号源全部添加  ─┐
                        ├──→ 步骤③ 场景预设（核心）
步骤② 显示屏配置完成  ─┘
                            ↓
                        步骤④ 系统设置
```

## 数据流向

```
Admin（产生数据）
  │ 配置接口下发
  ▼
解码拼控主机（存储 + 执行）
  │                    │
  │ 实时状态同步        │ 视频流
  ▼                    ▼
前端 Pad（只消费）     拼接大屏（物理显示）
  │
  │ 实时操作指令
  ▼
主机执行拼控动作
```

## 通信方式

| 类型 | 用途 |
|------|------|
| 配置接口 | Admin 配置下发、查询 |
| 实时通道 | 前端实时操作 + 状态同步 |
| 视频流协议 | 视频流传输 |

## 关键约束

1. 拼控逻辑在硬件，软件只发指令
2. 前端不产生数据，只消费后端预设
3. 场景 = 窗口布局 + 信号源绑定 + 轮巡策略，展示顺序属于场景列表属性
4. 配置有前置依赖：信号源+显示屏 → 场景 → 系统
5. 一个场景只有一组轮巡策略，前端只控制启停和筛选
