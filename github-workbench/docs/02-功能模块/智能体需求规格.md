# 运行中心显控系统 —— 智能体需求规格

> 版本：V1.0
> 适用对象：产品经理、技术负责人
> 方法论：垂域有限意图 + 规则/模型分流 + 置信度仲裁 + 安全职责分离

---

## 一、产品定位与使用场景

### 1.1 定位

智能体是运行中心显控系统的**语音控制层**，通过手持麦克风语音输入，让领导/操作员**无需动手点击**，直接通过自然语言对话完成大屏控制。

### 1.2 使用场景

| 场景 | 用户 | 典型表达 |
|------|------|---------|
| 领导参观 | 领导手持麦克风 | "切换到欢迎画面" / "放大看看201教室" |
| 日常操作 | 操作员 | "启动轮巡" / "把声音关掉" |
| 临时调整 | 操作员 | "把左上角的换成校长电脑" / "静音所有教室" |

### 1.3 输入方式

- **手持麦克风**：按住说话，松开后自动识别执行
- **无权限区分**：所有前端操作功能均可通过语音调用
- **本地部署**：大模型运行在本地服务器，无需联网

## 三、第二层：规则与模型分流

### 3.1 分流原则

| 分流类型 | 处理方式 | 适用意图 | 原因 |
|---------|---------|---------|------|
| **规则匹配** | 关键词/正则匹配，直接执行 | I04、I05、I08、I10、I13、I14、I15、I18 | 高频固定表达，100%可规则命中 |
| **规则+实体抽取** | 规则匹配意图，NLP/模型抽取实体 | I01、I02、I03、I06、I07、I09、I11、I12、I17 | 意图固定，但需识别具体目标（场景名、窗口位置、信号源名等） |
| **大模型理解** | 大模型语义理解+意图识别 | 模糊表达、多轮对话、指代消解、复合操作 | 自然语言过于灵活，规则无法覆盖 |

### 3.2 规则匹配意图（零成本，高速度）

| 意图 | 规则关键词 | 示例 |
|------|-----------|------|
| 启动轮巡 (I04) | "开始轮巡"、"启动轮巡"、"开始自动切换" | 用户说"开始轮巡" → 直接触发 |
| 停止轮巡 (I05) | "停止轮巡"、"暂停轮巡"、"停止切换" | 用户说"别轮巡了" → 直接触发 |
| 调节主音量 (I08) | "声音大一点"、"音量调到X" | 正则匹配"音量"+数字/百分比 |
| 全局静音 (I10) | "全部静音"、"都静音"、"取消静音" | 关键词匹配 |
| 退出反控 (I13) | "退出反控"、"返回"、"退出" | 关键词匹配（仅在反控模式下有效） |
| 查询当前场景 (I14) | "现在显示什么"、"当前是什么" | 关键词匹配 |
| 查询场景列表 (I15) | "有哪些场景"、"有什么模式" | 关键词匹配 |
| 查询运行日报 (I18) | "运行情况"、"日报"、"统计"、"报告" | 关键词匹配 |

### 3.3 规则+实体抽取意图（需识别实体）

| 意图 | 规则匹配意图 | 需抽取的实体 | 示例 |
|------|-------------|-------------|------|
| 场景切换 (I01) | 包含"切换"、"打开"、"切到" + 场景相关 | **场景名称** | "切换到智能巡课" → 意图：场景切换，实体：智能巡课 |
| 替换窗口信号源 (I02) | 包含"换"、"替"、"改成" + 窗口位置 | **窗口位置** + **信号源名称** | "把左上角换成201教室" → 窗口：左上角，信号源：201教室 |
| 放大展示 (I03) | 包含"放大"、"全屏"、"展示" + 信号源/窗口 | **信号源名称** / **窗口位置** | "放大201教室" → 信号源：201教室 |
| 筛选轮巡范围 (I06) | 包含"只轮巡"、"筛选" + 范围 | **轮巡范围** | "只轮巡教学楼A" → 范围：教学楼A |
| 窗口音量控制 (I07) | 包含"静音"、"声音" + 信号源/窗口 | **信号源名称** / **窗口位置** | "把201教室静音" → 信号源：201教室 |

| 查找定位信号源 (I17) | 包含"查找"、"定位"、"搜" + 信号源 | **信号源名称**（模糊匹配） | "查找201教室" → 信号源：201教室 |

### 3.4 大模型处理意图（模糊/复杂表达）

以下情况**必须走大模型**，规则无法覆盖：

| 情况 | 示例 | 大模型处理 |
|------|------|-----------|
| **模糊位置描述** | "把中间那个换成201教室" | 理解"中间"指哪个窗口 |
| **间接指代** | "刚才那个教室" | 结合对话历史，理解"刚才那个" |
| **复合操作** | "先切换到巡课模式，然后放大第一个窗口" | 拆解为多个意图（I01 + I03），按序执行 |
| **条件表达** | "如果有告警就切换到监控模式" | 理解条件逻辑 |
| **省略主语** | "静音"（没有说哪个） | 结合上下文推断目标 |
| **口语化变形** | "把那玩意儿换成201教室" | 理解"那玩意儿" = 当前选中窗口 |
| **模糊查找+后续操作** | "北教1的201教室在吗？在的话帮我找到，放大在墙上展示" | 拆解为 I17（查找定位）+ I03（放大展示），中间需处理歧义确认 |

## 四、第三层：实体识别与上下文

### 4.1 实体类型

| 实体类型 | 说明 | 示例值 |
|---------|------|--------|
| **场景名称** | 管理员预设的场景名称 | "智能巡课"、"接待欢迎"、"校领导驾驶舱" |
| **信号源名称** | 信号源名称（按命名规则） | "A教学楼-2楼-201-教师机位"、"校长电脑" |
| **窗口位置** | 回显画布中的窗口位置 | "左上角"、"第一个"、"左下角"、"中间" |
| **轮巡范围** | 轮巡筛选条件 | "教学楼A"、"电脑画面"、"全部" |
| **音量值** | 音量大小 | "50%"、"80"、"最大"、"最小" |
| **输出通道** | 声音输出通道 | "本机扬声器"、"音箱"、"HDMI输出" |
| **机位类型** | 摄像头机位 | "教师机位"、"学生机位"、"课件机位" |
| **教学楼/楼层/教室** | 结构树层级 | "A教学楼"、"2楼"、"201教室" |
| **键鼠指令** | 反控操作指令 | "下一页"、"点击确定"、"双击打开" |

### 4.2 信号源名称的模糊匹配

信号源名称按规则 **`教学楼-楼层-教室号-机位`** 命名，语音输入时支持**模糊匹配**：

| 用户说 | 匹配规则 | 匹配结果 |
|--------|---------|---------|
| "201教室" | 匹配名称中包含"201"的信号源 | A教学楼-2楼-201-教师机位、A教学楼-2楼-201-学生机位... |
| "A教学楼教师机位" | 匹配教学楼=A + 机位=教师机位 | A教学楼-1楼-101-教师机位、A教学楼-2楼-201-教师机位... |
| "2楼所有教室" | 匹配楼层=2 | 所有2楼教室的信号源 |

> **歧义处理**：模糊匹配到多个结果时，需走**仲裁机制**（追问确认）。

### 4.3 多轮对话与上下文

**示例对话链**：

```
用户：切换到智能巡课
AI：已切换到智能巡课模式

用户：放大第一个窗口
AI：已放大第一个窗口（结合上下文，知道是智能巡课场景的第一个窗口）

用户：把它换成201教室
AI：已将第一个窗口的信号源替换为A教学楼-2楼-201-教师机位（"它"指代第一个窗口）

用户：静音
AI：已将A教学楼-2楼-201-教师机位静音（省略主语，继承上下文）
```

**上下文继承规则**：
- 当前场景 → 后续操作默认针对当前场景
- 当前选中窗口 → "这个"/"它" 指代当前选中窗口
- 上一个操作的信号源 → 省略主语时继承上一个信号源
- 当前反控状态 → 反控模式下"下一页"等指令默认透传当前反控目标

## 五、第四层：置信度仲裁机制

### 5.1 双路并行判断

```
用户语音输入
    │
    ├─→ 规则引擎匹配 → 输出：意图 + 置信度A
    │
    └─→ 大模型语义理解 → 输出：意图 + 置信度B
    │
    ▼
  仲裁模块
    │
    ├─→ 高置信度（≥90%）→ 自动执行
    ├─→ 中置信度（60%~89%）→ 追问确认
    └─→ 低置信度（<60%）→ 拒绝执行，提示用户
```

### 5.2 仲裁规则

| 情况 | 规则置信度 | 模型置信度 | 仲裁结果 | 示例 |
|------|-----------|-----------|---------|------|
| 双高一致 | 95% | 97% | **自动执行** | "切换到智能巡课" → 双方都高，直接切 |
| 双高冲突 | 90% 场景切换 | 85% 轮巡启动 | **听模型+追问** | 追问"您是要切换场景还是启动轮巡？" |
| 单高单低 | 20% 未命中 | 75% 场景切换 | **听模型，执行** | "帮我调到监控模式" → 规则没命中，模型识别正确 |
| 双低 | 30% | 40% | **拒绝执行** | "今天天气怎么样" → 不属于显控功能，拒绝 |
| 歧义高 | 60% 替换201教室 | 55% 替换202教室 | **追问确认** | "请问是201教室还是202教室？" |
| 多机位歧义 | 80% 查找201 | 匹配到3个机位 | **追问确认** | "201教室有教师机位、学生机位、课件机位，请问您要哪个？" |

### 5.3 追问确认模板

| 场景 | 智能体回复 |
|------|-----------|
| 意图歧义 | "请问您是要切换场景还是启动轮巡？" |
| 实体歧义（信号源） | "请问是201教室还是202教室？" |
| 实体歧义（机位） | "201教室有教师机位、学生机位、课件机位，请问您要哪个？" |
| 目标歧义 | "请问要替换哪个窗口？" |
| 超出范围 | "这个操作不在语音控制范围内，请手动操作或联系管理员。" |
| 完全听不懂 | "抱歉，我没理解您的意思，请再说一遍。" |

### 5.4 特殊仲裁：多机位歧义

当用户语音触发 I17（查找定位信号源）或 I03（放大展示）时，模糊匹配到**同一教室的多个机位**，必须强制追问：

```
用户：放大201教室
AI：201教室有教师机位、学生机位、课件机位，请问您要放大哪个机位？

用户：教师机位
AI：已放大展示A教学楼-2楼-201-教师机位
```

> **注意**：即使规则置信度很高（如80%），只要存在多机位歧义，**必须追问确认**，不允许自动执行。

## 六、第五层：安全职责分离（护栏）

### 6.1 设计原则

AI只负责**理解意图和输出指令**，不负责**安全判断和执行确认**。

```
AI（意图识别）
    ↓ 输出：{intent, entity, params}
护栏系统（安全校验）
    ↓ 校验：权限、范围、风险
执行引擎（真正执行）
    ↓ 调用：设备API/场景API
```

### 6.2 护栏检查项

| 检查项 | 说明 | 触发条件 |
|--------|------|---------|
| **操作范围校验** | 校验目标场景/信号源/窗口是否存在 | 每次操作前 |
| **状态一致性校验** | 校验当前状态是否允许该操作 | 每次操作前 |
| **反控类型校验** | 校验反控目标是否为电脑信号源 | 进入反控时 |
| **重复操作拦截** | 拦截连续相同指令（防误触发） | 连续2次相同指令 |

### 6.3 高危操作（需二次确认）

以下操作属于**高风险**，即使AI识别成功，也需额外确认：

| 操作 | 确认方式 | 示例 |
|------|---------|------|
| 全局静音 | 语音确认 | "即将全部静音，请确认" → 用户说"确认"才执行 |
| 退出反控 | 无需确认（低风险） | 直接退出 |
| 场景切换 | 无需确认（低风险） | 直接切换 |
| 停止轮巡 | 无需确认（低风险） | 直接停止 |

> **说明**：显控平台前端操作本身风险较低（无删除、无破坏性操作），所以高危操作较少。但全局静音影响所有音频输出，需确认。

### 6.4 完全禁止的操作（AI绝不可执行）

| 禁止操作 | 原因 | AI响应 |
|---------|------|--------|

| 修改系统IP | 可能导致失联 | "该操作不在语音控制范围内。" |
| 格式化存储 | 破坏性操作 | "该操作不在语音控制范围内。" |


## 九、语音交互设计

### 9.1 唤醒与识别

| 环节 | 设计 |

### 9.2 语音播报模板

| 场景 | 播报内容 |
|------|---------|

- 音量调节 → 音量面板显示变化

```
手持麦克风（音频输入）
    │
    ▼

    │
    ▼

    │
    ▼

    │
    ▼
执行引擎（调用现有App API）
    │
    ▼

```

### 10.2 模型选型建议

| 模块 | 建议方案 | 原因 |
|------|---------|------|
| 语音识别（ASR） | 本地ASR引擎（如Paraformer、Whisper small） | 离线可用，隐私安全 |
| 意图识别（规则） | 本地关键词/正则匹配 | 零延迟，零成本 |

| 语音合成（TTS） | 本地TTS引擎（如ChatTTS、cosyvoice） | 离线可用，自然度高 |

## 十一、运行统计与日报模块

### 11.1 定位

运行统计与日报是智能体的**增值数据服务**，自动记录系统使用数据，定时生成统计报告，推送到Pad通知模块，供运维值班人员和产品经理查看。

### 11.2 目标用户

| 用户 | 关注点 | 使用方式 |
|------|--------|---------|
| **运维值班人员** | 系统运行状态、异常发现、使用频率 | 查看通知模块推送的日报 |
| **产品经理** | 功能使用热度、用户习惯、优化方向 | 查看日报/周报/月报/年报 |

### 11.3 推送形式

- **Pad通知模块**：在触控终端的通知中心推送文字版日报
- **语音播报（可选）**：用户可语音要求播报日报内容
- **大屏展示（可选扩展）**：在拼接屏上展示"今日运营数据"看板

### 11.4 触发方式

| 触发方式 | 说明 | 示例 |
|----------|------|------|
| **定时触发** | 每天固定时间自动生成并推送 | 每日9:00自动推送昨日日报 |
| **语音触发** | 用户语音主动要求查看运行情况 | "给我发下运行情况" / "看看今天的统计" |
| **手动触发** | 在通知模块中手动刷新查看 | 点击"生成日报"按钮 |

### 11.5 统计周期

| 周期 | 推送时间 | 统计范围 | 用途 |
|------|---------|---------|------|
| **日报** | 每日9:00 | 昨日0:00-23:59 | 日常运维监控 |
| **周报** | 每周一9:00 | 上周一0:00-周日23:59 | 周度运营分析 |
| **月报** | 每月1日9:00 | 上月1日0:00-月末23:59 | 月度产品分析 |
| **年报** | 每年1月1日9:00 | 上年全年 | 年度总结与规划 |

### 11.6 统计维度

#### 11.6.1 场景使用维度（核心）

| 统计项 | 说明 | 价值 |
|--------|------|------|
| **场景调用次数TOP5** | 每个场景被切换的次数 | 知道哪些场景最受欢迎 |
| **场景总使用时长** | 每个场景累计显示时长 | 判断场景"权重" |
| **场景切换路径** | 用户切换场景的典型路径 | 发现用户操作习惯 |
| **默认场景命中率** | 开机默认场景被保留/切换的比例 | 评估默认场景配置合理性 |

#### 11.6.2 信号源关注维度

| 统计项 | 说明 | 价值 |
|--------|------|------|
| **信号源查看次数TOP5** | 被放大展示/替换上墙的次数 | 哪些信号源最受关注 |
| **信号源搜索热词** | 用户在结构树/搜索框搜什么 | 发现用户关注焦点 |
| **机位偏好分布** | 教师机位/学生机位/课件机位的查看比例 | 了解用户关注点分布 |

#### 11.6.3 操作方式维度

| 统计项 | 说明 | 价值 |
|--------|------|------|
| **语音操作占比** | 语音操作次数 / 总操作次数 | 评估智能体使用率 |
| **语音操作成功率** | 成功执行 / 总语音指令 | 评估智能体识别质量 |
| **语音操作类型分布** | 场景切换/轮巡/音量/反控各占多少 | 哪些功能语音最常用 |
| **追问确认次数** | 仲裁触发"请确认"的次数 | 评估识别准确率 |
| **完全失败次数** | 识别失败/拒绝执行的次数 | 评估用户体验 |

#### 11.6.4 轮巡使用维度

| 统计项 | 说明 | 价值 |
|--------|------|------|
| **轮巡启动次数** | 每天启动轮巡的次数 | 轮巡功能使用频率 |
| **轮巡总运行时长** | 轮巡累计运行时间 | 轮巡功能活跃度 |
| **轮巡筛选偏好** | 全部/教学楼A/电脑画面的筛选比例 | 用户轮巡习惯 |

#### 11.6.5 时间维度

| 统计项 | 说明 | 价值 |
|--------|------|------|
| **使用高峰时段** | 每小时操作次数分布 | 知道什么时候最忙 |
| **连续操作间隔** | 两次操作之间的平均间隔 | 判断用户是"连续操作"还是"看一眼就走" |

#### 11.6.6 反控（KVM）维度

| 统计项 | 说明 | 价值 |
|--------|------|------|
| **反控进入次数** | 每天进入反控模式的次数 | 反控功能使用频率 |
| **反控目标分布** | 哪些电脑被反控最多 | 哪些电脑是"操作热点" |
| **反控平均时长** | 每次反控持续多久 | 用户是"看一眼"还是"深度操作" |

#### 11.6.7 异常与质量维度

| 统计项 | 说明 | 价值 |
|--------|------|------|
| **识别失败Top原因** | 为什么没听懂（噪声/口音/超出范围） | 优化ASR/模型 |
| **信号源无画面次数** | 哪些信号源频繁黑屏/无流 | 设备运维预警 |
| **用户重复指令** | 同一指令发了2次以上 | 可能是识别问题或系统延迟 |

### 11.8 语音交互设计

#### 11.8.1 语音触发日报

**用户**："给我发下运行情况" / "看看今天的统计" / "发下日报"

**智能体处理**：
1. 识别意图为 I18（查询运行日报）
2. 生成当日（或昨日）日报数据
3. 推送到Pad通知模块
4. **语音询问**："日报已生成，是否需要我播报？"

**用户回复**：
- "播报" → 智能体语音朗读日报关键内容
- "不用" / "不需要" → 智能体停止播报，用户可在通知模块查看
- 无回复（3秒）→ 默认不播报，用户自行查看

#### 11.8.2 语音播报内容（精简版）

当用户要求播报时，智能体只播报核心数据，不读完整日报：

> "今日运行统计：智能巡课场景使用23次，共320分钟。语音操作占比68%，成功率91%。使用高峰为上午9点和下午2点。无异常记录。"
