## 1. 设计概述

### 1.1 编写目的

本文档用于描述运行中心显控终端的硬件总体设计方案，明确产品定位、硬件架构、关键器件选型、接口规划、显示与触控能力、结构与部署要求、可靠性要求及样机验证方向。

本文档以《运行中心显控终端硬件需求PRD》为需求输入，以主板规格为硬件平台基础，结合运行中心拼接大屏控制场景，形成面向硬件研发、结构设计、工业设计、软件适配和测试验证的统一设计依据。

本文档不包含解码拼控主机、拼接大屏、摄像头、编码器等外部设备的硬件设计内容，仅描述安卓显控终端本体的硬件总体方案。

### 1.2 适用范围

本文档适用于运行中心显控终端两个版本的硬件设计、开发与验证工作。

| 版本 | 屏幕尺寸 | 屏幕比例 | 适用场景 |
| --- | ---: | ---: | --- |
| 标准版 | 27吋 | 16:9 | 中小型运行中心、标准操作台 |
| 宽屏版 | 34吋 | 21:9 | 中大型运行中心、宽幅操作台、大屏显控场景 |

两个版本采用统一的产品设计语言和硬件平台方案，仅在屏幕尺寸、整机结构、支架尺寸和显示比例上进行区分。

### 1.3 使用对象

运行中心显控终端的主要操作人员包括：

- 运行中心运维人员
- 值守人员
- 大屏操作人员
- 接待讲解人员

在领导接待、来宾参观、成果展示等场景中，学校领导或非技术用户也可进行低门槛触控操作，例如切换预设场景、查看大屏回显、调取指定画面等。

因此，本产品需要同时兼顾两类使用方式。

| 使用方式 | 主要用户 | 设计重点 |
| --- | --- | --- |
| 站姿触控操作 | 领导、讲解人员、运维人员 | 操作直观、一键触达、外观高级、触控舒适 |
| 坐姿精细操作 | 运维人员、配置人员 | 支持键鼠、便于配置、稳定可靠、接口充足 |

### 1.4 设计原则

1. **专业化**  
   产品不是普通显示器，而是运行中心拼接大屏的专用显控终端，应体现专业设备属性。

2. **易用性**  
   面向非技术用户时，应支持低门槛触控操作；面向运维人员时，应支持键鼠输入和精细配置。

3. **稳定性**  
   运行中心场景通常长时间不断电，终端需支持长期连续运行，满足显控软件稳定运行要求。

4. **一体化**  
   优先采用触控一体机形态，减少外接设备和桌面线缆暴露，提升运行中心操作台整体观感。

5. **可扩展性**  
   硬件接口需满足当前显控、回显、拉流、音频、维护需求，同时为后续语音交互、外设扩展、软件升级预留能力。

6. **场景适配**  
   结构、支架、接口、散热、线缆走向需适配运行中心操作台部署环境，兼顾站姿和坐姿使用。

## 2. 产品概述

### 2.1 产品定位

运行中心显控终端是面向运行中心、指挥中心、智慧校园展示空间的专用安卓触控终端。

该终端部署于运行中心操作台，用于运行显控软件，并通过显控软件对拼接大屏、电脑信号源、摄像头信号源、预设场景、回显画面和音频输出进行统一操作。

本产品不是普通办公显示器，也不是单纯的安卓平板，而是面向拼接大屏控制场景的一体化交互设备。其核心价值在于为运行中心提供稳定、直观、专业的大屏操作入口，使运维人员、讲解人员和非技术用户能够更便捷地完成大屏内容调度。

### 2.2 产品形态

显控终端优先采用触控一体机形态，整机由以下部分组成：

- 显示屏
- 触控模组
- RK3588 安卓主板
- 音频输出与拾音组件
- 外部接口模块
- 支架结构
- 电源适配系统
- 整机外壳与散热结构

产品分为两个版本。

| 项目 | 标准版 | 宽屏版 |
| --- | ---: | ---: |
| 屏幕尺寸 | 27吋 | 34吋 |
| 屏幕比例 | 16:9 | 21:9 |
| 本地显示目标 | 4K@60Hz | 4K@60Hz |
| 主要部署场景 | 中小型运行中心 | 中大型运行中心 |
| 使用姿态 | 站姿触控 / 坐姿操作 | 站姿触控 / 坐姿操作 |

两个版本应保持统一的外观风格、交互高度、接口规划和支架设计语言。

### 2.3 核心功能定位

显控终端硬件需支撑以下核心业务能力：

1. **大屏回显显示**  
   支持 1 路 HDMI 回显画面输入，目标能力为 4K@30Hz，用于显示拼接大屏当前画面。

2. **网络视频流显示**  
   支持 8 路 1080p 网络视频流在显控软件界面中同时显示。

3. **本地高分辨率显示**  
   本地显示能力目标为 4K@60Hz，保证显控界面、回显画面和信号源预览显示流畅。

4. **多点触控操作**  
   支持电容式多点触控，用于场景切换、信号源选择、拖拽上墙、窗口操作等交互。

5. **键鼠与维护外设接入**  
   支持 USB 键盘、鼠标、U盘等外设，用于后台配置、系统升级、日志导出和维护操作。

6. **音频输出**  
   支持将指定信号源音频输出至运行中心现有音响系统。

7. **语音输入预留**  
   具备拾音能力，为后续语音控制、智能体交互、语音切换场景等功能预留硬件基础。

8. **长期稳定运行**  
   满足运行中心长时间不断电、持续运行的使用要求。

### 2.4 典型应用场景

#### 2.4.1 站姿触控演示场景

适用于领导接待、来宾参观、智慧校园成果展示、运行中心讲解等场景。

用户站立在显控终端前，通过触控方式切换大屏场景、查看回显画面、调取指定内容。该场景要求设备外观高级、交互直观、触控舒适，并能与运行中心整体环境协调。

#### 2.4.2 坐姿运维配置场景

适用于运维人员进行信号源管理、场景配置、字幕设置、模板导入、系统升级、问题排查等操作。

用户通常坐在操作台前，通过键盘、鼠标和触控组合完成较为精细的配置操作。该场景要求接口充足、键鼠可用、显示清晰、系统稳定。

#### 2.4.3 大屏回显与信号调度场景

显控终端通过 HDMI 回显输入显示拼接大屏当前画面，同时在界面中显示多个网络视频流预览。用户可根据回显画面判断大屏显示状态，并通过显控软件完成信号源搜索、选择、拖拽、替换、放大等操作。

#### 2.4.4 运行中心长期值守场景

显控终端长期部署在运行中心操作台，平时处于持续运行状态。设备需具备良好的稳定性、散热能力、电源可靠性和接口耐用性，满足学校运行中心长期值守和日常操作要求。

### 2.5 产品边界

本产品硬件设计范围包括：

- 安卓显控终端整机
- RK3588 主板及接口适配
- 屏幕、触控、音频、拾音、电源、支架、结构和散热设计
- 与显控软件运行相关的硬件能力保障

本产品硬件设计范围不包括：

- 解码拼控主机硬件设计
- 拼接大屏硬件设计
- 摄像头、编码器等信号源设备设计
- 运行中心操作台家具设计
- 外部音响系统设计

### 3.5 部署环境输入

1. **操作台部署**  
   设备放置于运行中心操作台上，需与整体空间风格协调。

2. **长时间运行**  
   运行中心通常长时间不断电，终端需支持长期稳定运行。

3. **线缆可视性要求高**  
   接待、展示环境对桌面整洁度要求高，接口和走线需尽量隐藏。

4. **站姿和坐姿并存**  
   接待讲解时多为站姿触控；后台配置和维护时多为坐姿键鼠操作。

5. **外接设备较多**  
   可能接入网络、HDMI回显、音频输出、电源、键鼠、U盘等设备，需合理规划接口位置。

## 4. 系统总体架构

### 4.1 系统组成

运行中心显控终端作为运行中心显控系统的前端操作设备，与拼控主机、拼接大屏、信号源设备和外部音响系统共同构成完整运行环境。

| 组成部分 | 作用 |
| --- | --- |
| 显控终端 | 运行安卓显控软件，提供触控操作、回显显示、信号源预览、音频输出和维护接口 |
| 解码拼控主机 | 接收显控终端控制指令，执行视频解码、窗口拼接、大屏输出 |
| 拼接大屏 | 显示最终拼接画面 |
| 信号源设备 | 提供电脑画面、摄像头画面、平台画面等视频内容 |
| 编码器 | 将电脑HDMI等信号转为网络视频流 |
| 网络交换设备 | 提供显控终端、拼控主机、编码器、摄像头之间的网络连接 |
| 外部音响系统 | 播放显控终端输出的指定信号源音频 |
| 键鼠/U盘等外设 | 用于后台配置、维护、升级、日志导出等操作 |

### 4.3 与外部系统关系

显控终端不直接承担拼控主机的视频拼接任务，而是作为前端操作和显示终端，与外部系统协同工作。

| 外部对象 | 与显控终端关系 |
| --- | --- |
| 解码拼控主机 | 显控终端通过网络发送控制指令，拼控主机执行窗口切换、信号源替换、放大上墙等动作 |
| 拼接大屏 | 显控终端通过 HDMI 回显查看拼接大屏当前画面 |
| 摄像头 | 显控终端可通过网络接入部分视频流，用于本地预览 |
| 编码器 | 将电脑HDMI信号转为网络流，供显控软件预览或调度 |
| 电脑信号源 | 通过编码器和反控配置接入系统，用于电脑画面展示和远程操作 |
| 外部音响系统 | 显控终端将指定信号源音频输出至音响系统 |
| 运维外设 | 键盘、鼠标、U盘等通过USB接入终端，用于配置和维护 |

### 4.4 数据与信号流向

| 类型 | 流向 | 说明 |
| --- | --- | --- |
| 控制指令 | 显控终端 → 拼控主机 | 场景切换、信号源替换、放大、轮巡、反控等操作指令 |
| 回显信号 | 拼控主机 HDMI 环出 → 显控终端 HDMI IN | 显示拼接大屏当前完整合成画面 |
| 网络视频流 | 摄像头/编码器/平台 → 显控终端 | 用于底部预览区、信号源预览、局部视频显示 |
| 音频信号 | 显控终端 → 外部音响系统 | 输出当前监听的单路信号源音频 |

### 4.5 硬件设计边界

显控终端硬件设计需重点保障：

- 安卓显控软件稳定运行
- 1路 HDMI 回显画面显示
- 8路1080p网络流同时显示
- 4K@60Hz本地显示
- 电容触控交互
- 音频输出与拾音能力
- USB / 网络等维护接口
- 长时间运行可靠性
- 操作台部署和线缆隐藏

显控终端硬件不承担以下职责：

- 拼接大屏视频拼接合成
- 信号源统一解码输出到大屏
- 拼控主机内部板卡控制
- 摄像头、编码器、电脑主机等外部设备硬件能力

### 4.6 总体设计约束

| 约束项 | 说明 |
| --- | --- |
| 主控平台 | 固定采用 RK3588 |
| 内存与存储 | 当前确定为 8GB内存 + 64GB eMMC |
| 操作系统 | Android，版本随主板适配确定 |
| 本地显示 | 目标支持 4K@60Hz |
| HDMI回显 | 目标支持 1路 HDMI 回显 4K@30Hz，样机阶段验证 |
| 网络流显示 | 支持 8路1080p网络流同界面显示 |
| 产品形态 | 优先采用触控一体机 |
| 使用姿态 | 站姿触控优先，兼顾坐姿键鼠操作 |
| 电源设计 | 整机电源需覆盖主板、屏幕、背光、触控、音频和USB外设总功耗 |
| 显示接口 | 本机屏幕优先采用 HDMI 显示链路，eDP 作为备选 |

## 5. 硬件平台设计

### 5.1 主控平台选型

运行中心显控终端主控平台采用 DC-A588-V04 主板，核心处理器为瑞芯微 RK3588。

RK3588 是八核 64 位处理器，采用 4 × Cortex-A76 + 4 × Cortex-A55 架构，最高主频 2.2GHz，具备较强的图形显示、视频处理、网络通信和安卓应用运行能力，适合用于运行中心显控终端的多窗口界面、回显显示、网络视频预览和触控交互场景。

| 项目 | 设计选型 |
| --- | --- |
| 主控芯片 | RK3588 |
| 内存 | 8GB LPDDR4x |
| 存储 | 64GB eMMC |
| 操作系统 | Android，具体版本根据主板适配确定 |
| 使用场景 | 安卓显控软件运行、4K界面显示、HDMI回显显示、网络流预览、触控交互 |

### 5.2 存储设计

显控终端采用 64GB eMMC 作为系统存储，用于存放 Android 系统、显控软件、配置缓存、日志文件、升级包临时文件等。

显控终端不承担大规模本地录像存储任务，因此：

- 不配置本地大容量硬盘；
- 不提供 TF 卡口；
- 不以本地存储扩展作为核心能力；
- 长期运行过程中需控制日志、缓存、升级包残留，避免影响系统稳定性。

### 5.3 操作系统设计

终端运行系统为 Android，具体版本根据 DC-A588-V04 主板适配能力和显控软件兼容性确定。

| 项目 | 要求 |
| --- | --- |
| 启动方式 | 开机后自动进入显控软件或显控软件入口 |
| 运行模式 | 支持长时间前台运行显控应用 |
| 外设支持 | 支持触控、键盘、鼠标、U盘、HDMI输入、音频输出等外设 |
| 网络支持 | 支持有线网络优先，WiFi作为辅助或调试使用 |
| 系统维护 | 支持升级、日志导出、调试和恢复 |
| 稳定性 | 长时间运行不应出现应用异常退出、系统卡死、触控失效等问题 |

### 5.4 视频输入与显示能力

| 能力 | 设计目标 | 说明 |
| --- | --- | --- |
| 本地显示 | 4K@60Hz | 用于显示显控软件主界面 |
| HDMI回显输入 | 1路，目标4K@30Hz | 接收拼控主机 HDMI 环出画面，作为大屏回显 |
| 网络视频流预览 | 8路1080p | 用于信号源预览区域或局部视频窗口 |
| 多画面同时显示 | 支持 | HDMI回显和8路网络流需在同一界面中同时显示 |
| 对外HDMI输出 | 1路 | 用于扩展显示、调试显示或后续场景扩展 |

说明：

- 拼接屏当前合成画面由拼控主机通过 HDMI 环出输出，接入显控终端的 HDMI IN。
- 显控终端将该路 HDMI IN 画面作为“大屏回显”显示，用于操作人员确认拼接屏实际显示状态。
- HDMI回显是拼接屏最终合成后的完整画面，不是某一路单独信号源。
- 拼接合成由解码拼控主机完成，显控终端不承担拼接合成计算。
- HDMI回显 4K@30Hz 为设计目标，需在样机阶段结合主板、驱动和显控软件进行验证。
- 网络流预览用于前端操作辅助，不等同于拼控主机的大屏输出能力。

### 5.5 网络通信设计

显控终端后部提供双千兆网口。

| 接口 | 数量 | 位置 | 用途 |
| --- | ---: | --- | --- |
| 千兆网口 | 2 | 后部 | 控制通信、网络视频流、业务/管理网络 |

设计要求：

- 正式部署场景优先使用有线网络。
- 显控终端与拼控主机需网络可达，控制指令无明显延迟。
- 网络视频流拉取需保障带宽和稳定性。
- 双网口可用于业务网/管理网隔离，或用于现场网络冗余与扩展。
- WiFi不作为正式运行中心部署的主链路，除非现场网络条件明确允许。

### 5.6 对外接口设计

显控终端最终对外接口如下。

| 接口 | 数量 | 位置 | 用途 |
| --- | ---: | --- | --- |
| HDMI IN | 1 | 后部 | 接收拼控主机 HDMI 环出画面，作为大屏回显输入 |
| HDMI OUT | 1 | 后部 | 对外视频输出接口，可用于扩展显示、调试显示或后续场景扩展 |
| 千兆网口 | 2 | 后部 | 控制通信、网络视频流、业务/管理网络 |
| 音频口 | 1 | 后部 | 输出指定信号源音频至运行中心音响系统 |
| USB | 2 | 后部 | 固定接入键鼠、维护设备或其他外设 |
| USB | 2 | 前部 | 临时接入U盘、键鼠、维护设备 |
| 开关机按键 | 1 | 前部 | 开关机/唤醒 |

接口布局原则：

1. **长期接线后置**  
   HDMI IN、HDMI OUT、网口、音频口、电源等长期连接线缆应布置在后部，便于隐藏走线。

2. **临时维护前置**  
   前部 USB 口用于临时接入 U盘、键鼠等维护设备，方便现场操作。

3. **减少桌面线缆暴露**  
   后部接口应配合支架和结构设计隐藏线缆，保证运行中心操作台整洁。

4. **不提供TF卡口**  
   整机不提供TF卡扩展口，系统升级、日志导出和模板导入优先通过USB完成。

5. **不暴露Type-C作为用户接口**  
   若主板保留内部调试口或OTG能力，可作为产线调试或售后维护能力，不作为用户日常对外接口定义。

### 5.7 USB与维护接口设计

整机共提供4路对外USB接口：

- 后部 USB × 2
- 前部 USB × 2

| 用途 | 接口建议 |
| --- | --- |
| 键盘 | 后部USB |
| 鼠标 | 后部USB |
| U盘升级 | 前部USB |
| 日志导出 | 前部USB |
| 临时维护设备 | 前部USB |
| 备用外设 | 后部或前部USB |

说明：

- 后部USB适合长期接入键盘、鼠标等固定外设。
- 前部USB适合临时插入U盘、维护工具。
- USB供电能力需结合主板规格限制，整机设计中需避免多个高功耗USB外设同时使用导致供电不足。

### 5.8 音频与语音硬件设计

显控终端后部提供1路音频口，用于将指定信号源音频输出至运行中心现有音响系统。

| 功能 | 设计要求 |
| --- | --- |
| 音频输出 | 后部1路音频口，接入外部音响系统 |
| 本地扬声器 | 预留本地扬声器能力，非当前刚需 |
| 麦克风/拾音 | 需具备拾音能力，用于后续语音交互、语音控制等扩展 |
| 音量控制 | 通过软件音量控制为主，可结合硬件音量能力实现 |

说明：

- 运行中心多数场景下不依赖终端本地发声，优先保障外部音响输出。
- 本地扬声器作为未来外放提示音、系统提示、演示声音的功能预留。
- 拾音能力主要用于后续智能体交互、语音场景切换、语音控制拼接大屏等扩展能力。

### 5.9 电源与按键设计

显控终端前部提供开关机按键，用于整机开关机或唤醒操作。

电源设计需覆盖：

- RK3588 主板
- 27吋 / 34吋显示屏
- 屏幕背光
- 触控模组
- 音频功放 / 扬声器
- USB外设
- 风扇或散热组件
- 其他扩展外设

设计要求：

- 电源适配器不能仅按主板功耗选型，应按整机最大负载估算。
- 电源方案需预留功率余量。
- 电源接口应明确标识并具备防呆设计。
- 满足运行中心长期不断电使用要求。

## 6. 显示与触控设计

### 6.1 屏幕版本设计

| 项目 | 标准版 | 宽屏版 |
| --- | ---: | ---: |
| 屏幕尺寸 | 27吋 | 34吋 |
| 屏幕比例 | 16:9 | 21:9 |
| 本地显示目标 | 4K@60Hz | 4K@60Hz |
| 典型使用场景 | 中小型运行中心 | 中大型运行中心 |
| 产品形态 | 触控一体机 | 触控一体机 |

两个版本应保持统一的工业设计语言和交互逻辑。

### 6.2 本机屏幕显示链路

本机屏幕显示链路优先采用 HDMI 方案。

| 显示链路 | 定位 | 说明 |
| --- | --- | --- |
| 内部 HDMI 显示链路 | 优先方案 | 用于驱动27/34吋4K本机屏幕 |
| eDP显示链路 | 备选方案 | 根据屏幕模组和主板贴装情况评估 |
| LVDS显示链路 | 不建议作为4K主显示 | LVDS更适合1080p级别屏幕，不满足4K主显示目标 |

说明：

- 后部 HDMI OUT 是整机对外输出接口，不作为唯一的本机屏幕显示链路描述。
- 本机屏幕驱动方式需结合屏幕模组、驱动板、结构空间、线缆走向和主板接口最终确定。
- 若采用内部 HDMI 显示链路，需要确认屏幕驱动板供电、信号稳定性和结构固定方式。
- 若采用 eDP，需要确认主板贴装状态、屏幕兼容性和转接方案。

### 6.3 HDMI回显显示

显控终端通过后部 HDMI IN 接口接入大屏回显。

```text
拼控主机 / 大屏合成画面 HDMI 环出
        ↓
显控终端 HDMI IN
        ↓
显控软件回显区域显示
```

| 项目 | 要求 |
| --- | --- |
| 输入方式 | HDMI IN |
| 回显来源 | 拼控主机 HDMI 环出 |
| 回显内容 | 拼接屏最终合成后的完整画面 |
| 回显路数 | 1路 |
| 目标能力 | 4K@30Hz |
| 显示方式 | 在显控软件主界面中实时显示 |
| 用途 | 大屏状态确认、窗口选择、拖拽替换、放大上墙判断 |

说明：

- 回显画面是拼接屏当前显示结果，不是单路信号源。
- 回显用于辅助操作，不改变拼控主机输出逻辑。
- HDMI回显4K@30Hz需在样机阶段进行实际验证。

### 6.4 网络视频流显示

显控终端需支持8路1080p网络视频流同时显示，作为信号源预览和操作辅助能力。

| 项目 | 要求 |
| --- | --- |
| 视频路数 | 8路 |
| 单路分辨率 | 1080p |
| 显示方式 | 与回显画面在同一界面中同时显示 |
| 用途 | 信号源预览、目标信号源识别、拖拽上墙前确认 |
| 能力边界 | 本地预览能力，不代表拼控主机大屏输出能力 |

说明：

- 网络视频流通常来自摄像头、编码器或平台转发流。
- 8路1080p同时显示需结合安卓系统、硬件解码能力和显控软件播放器进行验证。
- 网络流预览画质可根据界面尺寸和性能策略采用主码流或子码流，具体由软件策略确定。

### 6.5 触控设计

显控终端优先采用电容式多点触控方案。

触控操作需支撑以下典型交互：

- 场景点击切换
- 信号源点击选中
- 信号源长按拖拽
- 回显窗口点击选中
- 弹窗按钮操作
- 搜索输入
- 列表滑动
- 音量、轮巡、反控等快捷控制

| 项目 | 要求 |
| --- | --- |
| 触控方式 | 电容式多点触控 |
| 触控稳定性 | 长时间运行后不应出现漂移、失灵、断触 |
| 拖拽体验 | 信号源拖拽应跟手、准确 |
| 站姿体验 | 不同身高用户站立操作时，触控角度应舒适 |
| 坐姿体验 | 运维人员坐姿操作时，支持精细点击与输入 |
| 抗误触 | 电源键、边缘区域和高频按钮需降低误触风险 |

### 6.6 站姿与坐姿适配

| 使用姿态 | 设计重点 |
| --- | --- |
| 站姿 | 屏幕角度适合站立观看和触控，常用操作一键触达，整机稳定不晃动 |
| 坐姿 | 支持键盘鼠标，便于配置、输入、维护和问题排查 |

支架角度建议：

- 站姿操作时，屏幕相对水平面的夹角建议支持 30°～60° 范围调节。
- 坐姿操作时，支架角度需兼顾运维人员查看和键鼠操作。
- 支架需保证触控操作时整机稳定，不应因点击、拖拽产生明显晃动。

## 7. 回显与视频能力设计

### 7.1 设计目标

显控终端的视频能力主要服务于显控软件前端操作，不承担拼接大屏的视频合成和输出任务。

| 能力 | 目标 |
| --- | --- |
| 大屏回显 | 接入1路 HDMI 回显画面，目标4K@30Hz |
| 网络视频预览 | 支持8路1080p网络视频流同时显示 |
| 本地界面显示 | 支持4K@60Hz显控软件界面 |
| 多画面同屏 | HDMI回显与网络视频流可在同一界面同时显示 |
| 操作辅助 | 支撑回显窗口选择、信号源预览、拖拽上墙、放大展示等交互 |

### 7.2 HDMI回显链路

HDMI回显链路用于将拼接屏当前实际显示画面同步到显控终端。

```text
拼控主机生成拼接屏最终合成画面
        ↓
拼控主机 HDMI 环出
        ↓
显控终端后部 HDMI IN
        ↓
显控软件回显区域显示
```

回显设计说明：

1. 回显来自拼控主机 HDMI 环出，是拼接屏当前完整合成画面。
2. 回显用于操作人员确认大屏当前实际显示内容，辅助完成窗口选择、目标替换、放大展示等操作。
3. 设计目标为 1 路 HDMI 输入，4K@30Hz。
4. 显控终端只显示回显，不负责拼接大屏的视频合成。
5. HDMI回显能力需在样机阶段验证，包括分辨率、帧率、延迟、稳定性和长时间显示效果。

### 7.3 网络视频流预览

网络视频流预览用于在显控终端本地显示信号源缩略图或局部视频画面，便于操作人员识别目标信号源。

| 项目 | 要求 |
| --- | --- |
| 同时显示路数 | 8路 |
| 单路分辨率 | 1080p |
| 视频来源 | 摄像头、编码器、平台转发流等 |
| 显示位置 | 信号源预览区、局部弹窗或辅助窗口 |
| 主要用途 | 信号源识别、预览确认、拖拽上墙前判断 |

视频流策略：

- 本地预览可根据界面尺寸和性能策略选择主码流或子码流。
- 底部缩略图区域可优先采用子码流或低分辨率预览流。
- 放大预览时可切换为更高分辨率码流。
- 8路1080p同时显示需结合播放器方案、硬件解码能力和网络条件验证。

### 7.4 多画面同屏显示

显控终端需支持 HDMI 回显与网络视频流在同一软件界面中同时显示。

典型界面组成包括：

- 中部大屏回显区域
- 底部信号源预览区
- 左侧信号源结构树
- 控制按钮栏
- 弹窗面板
- 状态提示区

| 要求 | 说明 |
| --- | --- |
| 同屏显示 | 回显画面和网络流预览可同时存在 |
| 界面流畅 | 页面切换、拖拽、弹窗、列表滚动无明显卡顿 |
| 视频稳定 | 回显和网络预览长时间显示不应黑屏、花屏、卡死 |
| 资源控制 | 多路视频显示时需控制CPU/GPU/内存占用 |
| 降级策略 | 网络或解码压力过高时，软件可降低预览流分辨率或帧率 |

### 7.5 视频能力边界

| 项目 | 是否由显控终端承担 | 说明 |
| --- | --- | --- |
| 拼接屏视频合成 | 否 | 由解码拼控主机完成 |
| 大屏窗口布局输出 | 否 | 由解码拼控主机完成 |
| 大屏信号源解码输出 | 否 | 由解码拼控主机完成 |
| HDMI回显显示 | 是 | 显控终端接收并显示回显画面 |
| 网络流本地预览 | 是 | 用于前端操作辅助 |
| 本地界面渲染 | 是 | 显控软件运行在终端本地 |
| 音频监听输出 | 是 | 输出当前选定信号源音频 |

### 7.6 样机验证要求

| 验证项 | 验证内容 |
| --- | --- |
| HDMI回显输入 | 4K@30Hz输入识别、显示稳定性 |
| 回显延迟 | 操作大屏后，终端回显变化延迟是否可接受 |
| 长时间回显 | 连续运行不少于24小时无黑屏、花屏、断流 |
| 8路1080p预览 | 同界面同时显示8路网络流是否稳定 |
| 回显+预览并发 | HDMI回显与8路网络流同时显示时系统是否稳定 |
| 触控拖拽并发 | 多视频显示时拖拽、点击、弹窗是否流畅 |
| 资源占用 | CPU、GPU、内存、温度是否在合理范围 |
| 异常恢复 | HDMI断开、网络流断开后恢复能力 |

## 8. 音频与语音设计

### 8.1 设计目标

显控终端音频与语音能力主要服务于两个方向：

1. **音频输出**  
   将指定信号源音频输出至运行中心现有音响系统，用于监听或现场播放。

2. **语音输入预留**  
   通过麦克风/拾音组件为未来语音交互、语音控制、智能体能力预留硬件基础。

### 8.2 音频输出设计

显控终端后部提供1路音频口，用于连接运行中心外部音响系统。

| 项目 | 要求 |
| --- | --- |
| 输出接口 | 后部1路音频口 |
| 输出对象 | 运行中心外部音响系统 |
| 音频来源 | 显控软件当前选定的信号源音频 |
| 控制方式 | 由显控软件控制当前监听对象和音量 |
| 使用方式 | 单路监听互斥，同一时间只输出一路信号源音频 |

说明：

- 显控软件中开启某路信号源音频后，系统将该路音频输出至外部音响。
- 同一时间只监听一路信号源，避免多路声音混杂。
- 用户可通过软件中的“一键静音”快速关闭当前监听。
- 后部音频口需保证长期接线稳定，适合接入运行中心现有音响系统。

### 8.3 本地扬声器预留

运行中心大多数场景下不依赖显控终端本地发声，因此本地扬声器不是当前核心刚需。

但从后续扩展考虑，整机设计建议预留本地扬声器能力，用于：

- 系统提示音
- 异常告警提示
- 本地演示声音
- 后续功能扩展

如果结构或成本受限，本地扬声器可作为选配或预留能力处理。

### 8.4 拾音能力设计

显控终端需具备拾音能力，用于未来语音交互扩展。

可能应用包括：

- 语音切换场景
- 语音调取信号源
- 语音控制拼接大屏
- 智能体交互
- 会议或讲解场景下的语音输入

| 项目 | 要求 |
| --- | --- |
| 拾音组件 | 内置麦克风或麦克风阵列，根据结构方案确定 |
| 使用场景 | 近场语音输入为主 |
| 软件接口 | Android系统可识别并调用 |
| 结构位置 | 避免被外壳、支架、桌面遮挡 |
| 抗干扰 | 尽量远离风扇、电源、高频信号线等干扰源 |

### 8.5 音频验证要求

| 验证项 | 验证内容 |
| --- | --- |
| 音频输出 | 是否可正常输出至外部音响 |
| 音量控制 | 软件音量调节是否有效 |
| 单路监听 | 切换不同信号源音频时是否互斥 |
| 一键静音 | 静音后外部音响是否立即停止输出 |
| 长时间播放 | 连续播放无杂音、断音、失真 |
| 拾音识别 | Android系统是否可识别麦克风输入 |
| 拾音效果 | 站姿/坐姿近场说话是否可被稳定采集 |

## 9. 结构与工业设计

### 9.1 设计目标

显控终端部署在运行中心操作台，是直接面向运维人员、讲解人员和部分领导用户的交互设备。因此，整机结构和工业设计不仅要满足硬件功能，还要体现专业、高级、简洁、易用的产品形象。

| 目标 | 说明 |
| --- | --- |
| 专业感 | 区别于普通办公显示器，体现运行中心专用显控终端属性 |
| 高级感 | 与运行中心、指挥中心、智慧校园展示空间匹配 |
| 易用性 | 支持站姿触控和坐姿键鼠操作 |
| 稳定性 | 触控、拖拽时整机不明显晃动 |
| 整洁性 | 接口和线缆尽量隐藏，保持操作台干净 |
| 可维护性 | 前部USB便于临时维护，后部接口便于长期接线 |

### 9.2 整机形态

显控终端优先采用触控一体机形态，整体形态参考“薄屏主体 + 后置支撑臂 + 横向底座”的结构方式。

整机组成包括：

- 显示屏和触控模组
- 前壳/后壳
- RK3588主板及接口板
- 音频与拾音组件
- 电源与散热模块
- 可调节支架
- 线缆隐藏结构

形态要求：

- 整机不应呈现普通显示器观感。
- 外观应简洁、科技感、未来感。
- 边框、支架、底座比例需协调。
- 不宜过于厚重或工业化。
- 正面应减少无关开孔和复杂装饰。
- 前部USB与开关机按键应整合在不破坏整体观感的位置。

### 9.3 支架设计

支架是显控终端结构设计的重点之一，需要同时满足站姿触控和坐姿运维两类场景。

| 项目 | 要求 |
| --- | --- |
| 调节方式 | 优先手动调节 |
| 角度范围 | 支持30°～60°范围调节 |
| 稳定性 | 触控屏幕任意位置时不明显晃动 |
| 操作便利性 | 调节过程简单，不需要复杂工具 |
| 线缆隐藏 | 支架需配合后部接口实现线缆收纳 |
| 外观一致性 | 支架设计语言与整机保持统一 |

说明：

- 站姿使用时，屏幕角度需适合不同身高用户触控。
- 坐姿使用时，屏幕角度需适合键鼠配置和近距离查看。
- 支架底座需具备足够重量或结构稳定性，避免拖拽操作时整机位移。

### 9.4 接口布局与走线设计

整机接口按“长期接线后置、临时维护前置”的原则布局。

后部接口：

- HDMI IN
- HDMI OUT
- 双网口
- 音频口
- 后部USB × 2
- 电源接口

前部接口：

- 开关机按键
- 前部USB × 2

设计要求：

1. **后部线缆隐藏**  
   长期连接的 HDMI、网线、音频线、电源线应通过后部或支架结构隐藏走线，减少桌面线缆暴露。

2. **前部维护便利**  
   前部USB用于临时接入U盘、键鼠或维护设备，应便于操作人员快速插拔。

3. **接口标识清晰**  
   接口需有清晰标识，便于现场安装和售后维护。

4. **避免误插**  
   HDMI IN 和 HDMI OUT 应通过位置、标识或颜色区分，避免安装时接反。

5. **接口强度**  
   后部长期连接接口需具备良好的固定和抗拉拽能力。

### 9.5 散热设计

显控终端需满足长时间运行要求，散热设计需覆盖主板、屏幕背光、电源模块等主要热源。

主要热源包括：

- RK3588主板
- 屏幕背光
- 电源适配/电源转换模块
- 音频功放
- USB外设供电
- 可能存在的风扇或散热组件

| 原则 | 说明 |
| --- | --- |
| 长时间稳定 | 满足运行中心长期不断电使用 |
| 低噪声 | 运行中心接待和展示场景下不应产生明显噪声 |
| 不影响外观 | 散热孔位应与外观设计协调 |
| 不影响触控 | 长时间运行后前面板触摸区域温升应可接受 |
| 易维护 | 如使用风扇，应考虑灰尘、寿命和后期维护 |

建议在样机阶段进行温升测试，重点关注高亮度显示、HDMI回显、8路网络流同时显示、长时间运行条件下的温度表现。

### 9.6 材料与外观设计

整机外观颜色确定为银色，整体风格应体现科技感、专业感和高级感，与运行中心操作台、拼接大屏和现代化展示空间相匹配。

设计要求：

- 整机主色采用银色。
- 表面质感建议采用细腻哑光或半哑光金属质感，避免高反光镜面影响观感。
- 前面板尽量简洁，减少复杂装饰。
- 边框宽度应控制在合理范围，避免厚重。
- 支架与底座颜色、材质、工艺应与主机保持一致。
- 外壳表面应耐指纹、易清洁。
- 前部USB和开关机按键位置需兼顾美观与实用。
- 底座扬声器开孔需排列规整，避免影响整机高级感。
- 接口区域应尽量隐藏，减少线缆外露。

### 9.7 结构验证要求

| 验证项 | 验证内容 |
| --- | --- |
| 支架稳定性 | 站姿触控、拖拽时整机不明显晃动 |
| 角度调节 | 30°～60°范围调节是否顺畅、可靠 |
| 线缆隐藏 | 后部线缆接入后桌面是否整洁 |
| 接口插拔 | 前后USB、HDMI、网口、音频口插拔是否方便 |
| 接口误插 | HDMI IN/OUT标识是否清晰 |
| 散热表现 | 长时间运行温升是否满足要求 |
| 外观一致性 | 标准版和宽屏版是否保持统一设计语言 |
| 维护便利性 | U盘升级、日志导出、键鼠接入是否方便 |

## 10. 电源与可靠性设计

### 10.1 电源设计目标

显控终端需满足运行中心长期部署和长时间运行要求。整机电源设计不能仅依据主板功耗选型，而应综合考虑主板、屏幕、背光、触控、音频、USB外设和散热组件的总功耗。

| 项目 | 要求 |
| --- | --- |
| 主板供电 | DC 12V输入 |
| 整机供电 | 外置电源适配器供电 |
| 供电对象 | 主板、屏幕、背光、触控、音频、USB、散热组件 |
| 功率余量 | 需预留外设接入和长时间运行余量 |
| 使用场景 | 运行中心长期不断电运行 |
| 安全性 | 电源接口明确标识，避免误插和接触不良 |

说明：主板建议电源规格为 12V 2-5A，但该参数仅覆盖主板及部分外设需求。27吋和34吋整机包含大尺寸屏幕和背光系统，整机适配器功率需在结构样机和电气评估阶段单独核算。

### 10.2 电源接口与走线

电源接口建议布置在整机后部，配合支架和底座进行隐藏走线。

设计要求：

1. 电源线应从后部或底座区域引出，避免在操作台正面暴露。
2. 电源接口应具备明确标识。
3. 电源接口应有足够插拔强度，避免长期使用后松动。
4. 电源线与 HDMI、网线、音频线等长期线缆应统一收纳。
5. 若条件允许，可在底座或支架中预留走线通道。

### 10.3 长时间运行可靠性

| 项目 | 要求 |
| --- | --- |
| 系统稳定 | 长时间运行显控软件不崩溃、不死机 |
| 显示稳定 | 本地屏幕长时间显示无黑屏、闪烁、花屏 |
| 回显稳定 | HDMI回显长时间显示稳定 |
| 网络稳定 | 与拼控主机通信稳定，网络流预览不中断或可恢复 |
| 触控稳定 | 长时间运行后触控不漂移、不失灵 |
| 音频稳定 | 音频输出无明显杂音、断音 |
| 温升可控 | 满载运行时外壳和触控区域温度可接受 |

### 10.4 散热可靠性

主要热源包括：

- RK3588主板
- 屏幕背光
- 电源转换模块
- 音频功放
- USB外设供电
- 显示驱动板
- 散热组件自身

散热设计要求：

1. 满足高亮度显示、HDMI回显、8路网络流同时显示条件下的温升控制。
2. 散热结构不应破坏整机外观。
3. 散热孔不应被支架、底座或操作台遮挡。
4. 若采用风扇，应优先低噪声方案，避免影响运行中心接待和讲解。
5. 需考虑灰尘堆积后的散热衰减和维护便利性。

### 10.5 接口可靠性

| 接口 | 可靠性要求 |
| --- | --- |
| HDMI IN | 长期接入回显线缆，需稳定识别信号，避免松动 |
| HDMI OUT | 对外扩展或调试输出，需标识清晰 |
| 双网口 | 长期连接，需具备良好插拔寿命和卡扣稳定性 |
| 音频口 | 长期连接外部音响系统，避免接触不良 |
| 后部USB | 支持键鼠等长期接入 |
| 前部USB | 支持频繁插拔U盘和维护设备 |
| 电源接口 | 防松动、防误插，满足长期运行 |

### 10.6 外观与结构可靠性

整机采用银色外观，形态参考“薄屏主体 + 后置支撑臂 + 横向底座”的结构方式。

结构可靠性要求：

- 薄屏主体应保证足够结构强度。
- 后置支撑臂应支撑不同角度下的屏幕重量。
- 横向底座需具备足够稳定性和配重能力。
- 站姿触控、拖拽操作时整机不应明显晃动。
- 支架长期调节后不应出现松垮、异响或角度保持失效。
- 底座扬声器开孔、散热孔和走线结构需兼顾外观和功能。

## 12. 风险与待确认项

### 12.1 硬件平台风险

| 风险项 | 说明 | 应对措施 |
| --- | --- | --- |
| HDMI回显4K@30Hz稳定性 | 主板规格支持4K输入，但实际显示效果需验证 | 样机阶段进行长时间回显测试 |
| 8路1080p并发显示 | 多路网络流对CPU/GPU/内存有压力 | 软件播放器优化，必要时采用子码流或降低帧率 |
| Android长期运行稳定性 | 长时间运行可能出现内存泄漏、应用异常 | 进行长时间稳定性测试和系统看门狗策略 |
| 整机散热 | 大屏幕+主板+视频解码可能产生较高热量 | 样机阶段进行满载温升测试 |
| USB供电 | 多个外设接入可能导致供电压力 | 限制高功耗外设，验证USB供电能力 |

### 12.2 显示与结构风险

| 风险项 | 说明 | 应对措施 |
| --- | --- | --- |
| 本机4K显示链路 | HDMI/eDP/驱动板方案需和屏幕模组匹配 | 优先验证HDMI显示链路，eDP作为备选 |
| 宽屏版UI适配 | 34吋21:9界面与标准版差异较大 | 软件需适配不同屏幕比例 |
| 支架稳定性 | 大尺寸屏幕站姿触控时可能晃动 | 加强底座配重和支撑臂结构 |
| 线缆隐藏 | 接口较多，后部线缆容易杂乱 | 支架和底座预留走线通道 |
| 银色外观工艺 | 银色表面可能出现色差、划痕、反光 | 确认表面处理工艺和样件效果 |

### 12.3 功能边界风险

| 风险项 | 说明 | 应对措施 |
| --- | --- | --- |
| 用户误认为终端承担拼控 | 显控终端只负责操作、回显和预览 | 文档和培训中明确拼控由拼控主机完成 |
| HDMI IN/OUT误接 | 两个HDMI接口用途不同 | 接口丝印、颜色或布局区分 |
| 前部USB误用 | 频繁插拔可能影响外观或接口寿命 | 选用耐插拔接口，结构加强 |
| 音频输出场景差异 | 不同运行中心音响接口不同 | 预留转接方案或适配线材 |

### 12.4 待确认项

| 待确认项 | 当前状态 |
| --- | --- |
| 标准版/宽屏版具体屏幕型号 | 待结构和供应链确认 |
| 本机屏幕最终显示链路 | 优先HDMI，待样机验证 |
| 整机电源适配器功率 | 待整机功耗评估 |
| 本地扬声器是否标配 | 当前按预留能力处理 |
| 麦克风形态 | 内置麦克风或麦克风阵列待结构确认 |
| 散热方式 | 被动散热或主动散热待样机验证 |
| 银色表面处理工艺 | 待工业设计和样件确认 |
