## 一、红苹果——集中式解码拼控架构

### 1.1 整体数据流

```
                    信号源层                      网络传输层                    拼控处理                    输入层
        ┌──────────────────────┐    ┌──────────────────────┐    ┌──────────────────────┐    ┌──────────────┐
        │  RTSP 摄像头 ×N      │───▶│                      │    │                      │    │              │
        │                      │    │                      │    │  红苹果解码拼控主机   │    │  RTSP 输出流  │
        │  HDMI 信号源          │    │   千兆局域网交换机    │───▶│                      │───▶│  （合成画面） │
        │  （经编码器→RTSP）    │───▶│                      │    │  RTSP解码→电信号处理  │    │              │
        │                      │    │                      │    │  矩阵切换→画面合成    │    │  HDMI→大屏   │
        │  电脑桌面             │    │                      │    │  再编码→hdmi输出      │    │              │
        │  （经编码器→RTSP）    │───▶│                      │    │                      │    │  移动端拉流   │
        └──────────────────────┘    └──────────┬───────────┘    └──────────┬───────────┘    └──────────────┘
                                               │                           │
                                               │  PENET 控制指令           │
                                               │◀──────────────────────────│
                                               │                           │
                                        ┌──────┴───────┐          ┌───────┴───────┐
                                        │ RK3588 终端   │          │  AP（手机/平板）│
                                        │ App 控制台     │          │  移动端     │
                                        └──────────────┘          └───────────────┘
```

### 1.2 数据流详解

| 阶段 | 信号形态 | 传输介质 | 说明 |
|:--|:--|:--|:--|
| **输入** | **RTSP 流** | 千兆以太网 → 交换机 | 所有信号源（摄像头/电脑/HDMI设备）统一编码为 RTSP，接入交换机 |
| **汇聚** | IP 包 | 交换机 | 交换机汇总所有 RTSP 流，转发给拼控主机 |
| **解码** | RTSP → **电信号（基带）** | 拼控主机内部 | 拼控主机将所有 RTSP 流解码还原为基带电信号，送入矩阵处理 |
| **处理** | 电信号 | 内部矩阵 | 信号切换、画面合成、拼接、叠加、漫游 |
| **再编码** | 电信号 → **RTSP** | 拼控主机内部 | 处理后的合成画面重新编码 |
| **输出** | HDMI | 直连 | 回显画面 |
| **控制** | PENET 协议（TCP） | 交换机 | RK3588 终端通过 PENET 指令控制拼控主机的矩阵切换和画面布局 |

### 1.3 核心特点

| 特点 | 说明 |
|:--|:--|
| **全 IP 化输入** | 包括 HDMI 信号源也经编码器转为 RTSP 再入网 |
| **集中式处理** | 拼控主机是唯一处理节点，所有 RTSP 流汇聚于此 |
| **双形态输出** | 处理后同时输出 HDMI（APP回显）+ HDMI（直连大屏） |
| **私有控制协议** | PENET 协议控制拼控行为，当前仅支持"上墙"和"转预置位"两种联动动作 |
| **RK3588 承载业务** | Android 终端承担前端 UI + AI 推理 + PENET 指令下发 |

### 1.4 关键瓶颈

| 瓶颈 | 影响 |
|:--|:--|
| **单点故障** | 拼控主机是整个系统的唯一点，出问题则全部瘫痪 |
| **PENET 协议受限** | 仅 2 种联动动作，无法实现复杂场景联动 |
| **解码并发上限** | 受拼控主机板卡数量限制，输入信号路数有上限 |
| **扩展性差** | 增加信号源需加输入板卡，受机箱槽位限制 |

## 附录

### 5.1 网络基础概念速查

| 概念 | 解释 |
|:--|:--|
| **AP（Access Point）** | 无线接入点，将 WiFi 设备桥接进入有线局域网，无线设备拿到同一子网 IP |
| **信道层碰撞** | 多个 AP 使用同一 WiFi 频道导致信号互相干扰 |
| **IP 冲突** | DHCP 分配相同 IP 给两台设备，或静态 IP 和 DHCP 池重叠 |
| **跨网段通信** | 需三层交换机/路由器做网关转发，默认不通 |
| **RTSP 拉流** | 标准视频流协议，手机 VLC 等播放器输入 `rtsp://IP:端口/路径` 即可观看 |

### 5.2 推荐网络部署方案

- **静态 IP + MAC 绑定**：设备端保持 DHCP 自动获取，在路由器上绑定 MAC ↔ IP
- **固定 IP 与 DHCP 池不重叠**：如固定用 192.168.1.10-99，DHCP 池用 .100-.200
- **显控设备同网段**：避免跨网段路由增加延迟
