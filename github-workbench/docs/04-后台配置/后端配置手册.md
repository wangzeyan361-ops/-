一、系统概述

### 1.1软件与硬件分工

运行中心显控系统采用软件控制+硬件执行的分层架构：

| 层级 | 角色 | 负责内容 | 对应设备 |
| --- | --- | --- | --- |
| 软件层 | 本配置系统 | 信号源管理、场景预设、屏幕配置、系统设置 | 安卓触控终端 |
| 硬件层 | 解码拼控主机 | 视频解码、画面拼接、信号输出、RTSP拉流 | 机房拼控主机 |
| 显示层 | 拼接大屏 | 物理显示画面 | 墙上拼接大屏 |

核心原则：所有配置数据在软件层生成，通过协议下发到拼控主机执行。软件只负责配置与指令，不处理视频流。

注意：安卓端必须能访问拼控主机主控IP，否则App无法连接拼控主机。外接网口用于视频流拉取，可按信号源网络规划配置到不同网段。

### 1.2配置流程总览

后端配置的整体流程遵循先基础、后场景的顺序：

步骤1：硬件IP配置完成
步骤2：登录管理员界面
步骤3：主控与外接网口配置
步骤4：信号源管理
步骤5：显示屏配置
步骤6：场景预设（核心）
步骤7：系统设置（网络/密码/运维）

提示：硬件IP配置是所有后续操作的前置条件，必须在App配置前完成。

### 1.3配置顺序与依赖关系

| 模块 | 依赖前置模块 | 说明 |
| --- | --- | --- |
| 硬件IP配置 | 无 | 最先完成，确保硬件网络就绪 |
| 登录与界面 | 硬件IP配置 | 确保能正常连接硬件 |
| 主控与外接网口配置 | 硬件IP配置 | 填写主控IP，按需配置外接网口IP，建立硬件对接 |
| 信号源管理 | 主控与外接网口配置 | 信号源通过RTSP拉流，需硬件网络就绪 |
| 显示屏配置 | 无 | 与信号源管理可并行配置 |
| 场景预设 | 信号源管理+显示屏配置 | 场景需要引用信号源和屏幕布局 |
| 系统设置 | 无 | 随时可配，建议最后统一检查 |

配置顺序建议：

- 通过ipconfig工具配置拼控主机和编码器的固定IP
- 打开App，登录管理员界面
- 进入「主控与外接网口配置」，配置硬件对接信息
- 进入「信号源管理」，添加全部信号源
- 进入「显示屏配置」，设置屏幕布局
- 进入「场景预设」，创建场景并绑定信号源
- 进入「系统设置」，配置安卓端网络、密码、升级等
## 二、准备工作（硬件IP配置）

在App配置之前，必须先在硬件层面完成网络IP配置，确保拼控主机、编码器、安卓终端处于同一网段且网络互通。

### 2.1解码拼控主机IP设置

拼控主机的固定IP需通过外部的ipconfig工具进行设置。

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 打开ipconfig工具 | 打开外部网络配置工具 |
| 2 | 设置固定IP地址 | 如192.168.1.100，与后续安卓端要设置在同网段 |
| 3 | 设置子网掩码 | 如255.255.255.0 |
| 4 | 设置网关 | 如192.168.1.1 |
| 5 | 保存并重启网络 | 确认IP生效，网络连通 |

提示：拼控主机主控IP设置后请牢记，后续在App「主控与外接网口配置」中需要填写该IP。

### 2.2编码器（编码盒子）IP设置

编码器（编码盒子）的IP同样需通过外部的ipconfig工具，通过Web界面设置编码参数。

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 设置固定IP地址 | 如192.168.1.101，与拼控主机同网段 |
| 2 | 登录编码器管理界面 | 通过浏览器访问编码器默认IP |
| 3 | 配置编码参数 | 分辨率、码率 |
| 4 | 记录RTSP输出地址 | 如rtsp://192.168.1.101:554/stream1 |

提示：每个编码器的IP必须唯一，并与负责拉取该路视频流的拼控主机外接网口网络可达。记录所有编码器的IP和RTSP地址，后续信号源配置时需要用到。

## 三、登录与界面

### 3.1管理员入口

打开「运行中心显控系统」App，在登录界面点击右上角「管理员入口」：

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 点击右上角「管理员入口」 | 切换至管理员登录模式 |
| 2 | 输入6位数字密码 | 初始密码由系统预设，可在系统设置中修改 |
| 3 | 点击「进入系统」 | 进入后端配置界面 |

提示：若忘记管理员密码，需联系厂商技术支持或通过恢复出厂设置重置。

### 3.2配置界面布局

管理员界面采用左侧导航栏+右侧内容区的经典布局。界面分为四大功能区域，通过左侧导航栏切换：

| 导航项 | 内容 |
| --- | --- |
| 信号源管理 | 添加、编辑、删除信号源；配置摄像头画面、电脑画面、平台同步、字幕叠加和反控参数 |
| 显示屏配置 | 设置屏幕物理布局（行列数）；端口映射绑定；错接校正 |
| 场景预设 | 创建/编辑场景；设置场景布局；信号源绑定；轮巡策略；前端展示顺序 |
| 系统设置 | 终端IP、密码管理、版本升级、存储日志、初始化恢复 |

## 四、主控与外接网口配置（硬件对接）

用于配置安卓端与解码拼控主机的硬件对接信息。必须在信号源管理之前配置。

### 4.1主控IP配置

系统只有一个主控卡，管理员需要配置的是主控IP。主控IP用于安卓端与解码拼控主机建立控制通信。

| 配置项 | 说明 | 示例 |
| --- | --- | --- |
| 主控IP | 解码拼控主机主控卡的固定IP地址（通过ipconfig设置） | 192.168.1.100 |

注意：此处填写的IP必须与2.1节中通过ipconfig工具设置的拼控主机主控IP一致。板卡之间有内部通信IP，不需要管理员在本页面配置。

### 4.2外接网口与交换机配置（大规模信号源拉流）

当需要接入大量信号源时，可通过外接多根网线并接入交换机，分担不同信号源的网络带宽压力。本质是网络带宽规划，不是对板卡内部IP进行配置。

| 配置项 | 说明 | 示例 |
| --- | --- | --- |
| 外接网口IP1 | 拼控主机外接网口的IP地址，用于接入对应交换机或信号源网段 | 192.168.10.100 |
| 外接网口IP2 | 另一条外接网线对应的IP地址，可接入另一组信号源网段 | 192.168.20.100 |
| 外接网口IPN | 按实际外接网口数量配置 | 192.168.30.100 |

注意：

- 主控IP用于控制通信，外接网口IP主要用于拉取信号源视频流

- 外接网口需与对应交换机下的摄像头、编码器处于可达网段

- 板卡内部通信IP由设备内部维护，不在后台页面中配置

- 多根网线的目的在于通过交换机分流信号源带宽，避免大量RTSP流集中在单一路径上
## 五、信号源管理

### 5.1添加信号源（RTSP接入/平台同步）

本系统统一以网络流形式管理视频信号源。信号源类型包括摄像头画面、电脑画面等，其中摄像头可直接通过RTSP接入，电脑画面需通过编码盒子转码为RTSP流后接入。

说明：HDMI信号源（如电脑画面）需先通过编码盒子转码为RTSP流后，再填入对应的RTSP地址接入。电脑画面若需要支持前端反控，还需配置被控电脑的IP、用户名和密码。

信号源命名规则：

信号源名称建议采用固定格式：楼宇-楼层-教室号-机位。若需要展示课程、教师等信息，可在字幕或扩展字段中维护。

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| 楼宇 | 思政楼/北教 | 教学楼或区域名称 |
| 楼层 | 1层/2层/3层 | 楼层名称 |
| 教室号 | 101/202 | 教室或房间号 |
| 机位 | 老师机位/学生机位/课件机位/电脑画面 | 信号源位置或类型描述 |

命名示例：思政楼-2层-201-老师机位、北教-1层-101-电脑画面

提示：结构树优先按楼宇、楼层、教室号解析。命名不规范会影响前端结构树搜索和定位效果。

信号源添加方式：

| 方式 | 适用场景 | 操作说明 |
| --- | --- | --- |
| 表格导入 | 批量添加大量信号源 | 下载模板表格→按命名规则填写信号源信息→上传导入 |
| 手动新增 | 单个或少量信号源添加 | 点击「添加信号源」→填写表单→保存 |
| 平台同步 | 从已有平台获取信号源列表和课表相关数据 | 点击「从平台同步」→选择对接平台→自动获取并导入 |

手动新增步骤：

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 点击「添加信号源」按钮 | 新增行 |
| 2 | 选择信号源类型 | 摄像头画面或电脑画面 |
| 3 | 填写信号源名称 | 按命名规则填写，如思政楼-2层-201-老师机位 |
| 4 | 填写RTSP地址 | 如rtsp://192.168.1.101:554/stream1。若接入的是编码盒子转码的电脑画面，则填写对应编码盒子的RTSP地址（见2.2节） |
| 5 | 点击「保存」 | 系统尝试拉流验证，验证通过后保存 |

提示：RTSP地址格式通常为rtsp://IP:端口/通道。IP地址为信号源设备（摄像头或编码盒子）的实际IP。若不确定，可查看设备Web管理界面或咨询设备厂商。

### 5.2HDMI信号源接入说明

本系统不直接接入HDMI信号。HDMI信号源（如电脑、播放器等）需通过编码盒子（视频编码器）先转码为RTSP流，再按5.1节的步骤统一添加。

接入流程：

HDMI信号源（电脑）
│
▼HDMI线缆
编码盒子（视频编码器）
│将HDMI信号编码为RTSP流
▼
填写编码盒子的RTSP地址到本系统
│如rtsp://编码盒子IP:554/stream1
▼
系统统一通过RTSP协议拉流处理

编码盒子配置：

- 编码盒子需与拼控主机在同一网络内
- 在编码盒子管理界面配置编码参数（分辨率、码率、编码格式等）
- 记录编码盒子输出的RTSP地址，填入本系统的信号源配置中
反控配置：

| 配置项 | 说明 |
| --- | --- |
| 电脑IP | 被控电脑的IP地址，需与Pad/服务端网络可达 |
| 用户名 | 被控电脑远程登录用户名 |
| 密码 | 被控电脑远程登录密码 |

提示：当信号源类型选择为「电脑画面」时，该信号源可配置软件反控能力。反控方式类似远程桌面/向日葵一类的软件层远控，不依赖摄像头信号源。未配置电脑IP、用户名、密码时，前端不应进入反控。

### 5.3信号源分组与结构树配置

系统根据信号源名称自动生成分组结构树，无需手动配置层级关系。

自动生成规则：

系统按照名称格式楼宇-楼层-教室号-机位自动解析并生成树状结构：

└──思政楼
├──1层
│├──101教室
││├──思政楼-1层-101-老师机位
││├──思政楼-1层-101-学生机位
││└──思政楼-1层-101-课件机位
│├──102教室
││└──思政楼-1层-102-老师机位
│└──...
├──2楼
│├──201教室
││└──A-2-201-老师机位
│└──...
└──3楼
└──...

结构树特点：

- 自动解析：根据名称中的连字符-自动分层，无需手动拖拽绑定
- 实时更新：新增或删除信号源后，结构树自动刷新
- 一键搜索：前端操作员可通过结构树快速定位信号源
提示：命名规则建议遵循楼宇-楼层-教室号-机位格式，否则结构树解析可能异常。层级建议控制在4层以内，确保前端操作员查找效率。

### 5.4字幕叠加

支持在信号源画面上叠加文字标注，如教室名称、时间等。

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 在信号源列表中选择目标信号源 | - |
| 2 | 打开字幕叠加配置 | 进入该信号源的字幕配置项 |
| 3 | 输入每路信号源字幕内容 | 如「北教101-单片机-刘教授」 |
| 4 | 设置全局字幕位置 | 左上/右上/左下/右下/居中 |
| 5 | 设置全局字幕字体大小、颜色 | 推荐白色字体+黑色半透明背景，确保各种画面下可见 |
| 6 | 点击「保存」 | 字幕实时叠加到该信号源画面上 |

提示：字幕叠加仅影响该信号源在拼接屏上的显示，不影响原始视频流。每个信号源可独立配置字幕。字幕位置，字体大小，颜色全局有效。

## 六、显示屏配置

### 6.1物理布局设置

定义拼接大屏的物理排列方式，建立数字孪生模型。

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 进入「显示屏配置」 | - |
| 2 | 设置行数 | 大屏拼接的行数，如2行 |
| 3 | 设置列数 | 大屏拼接的列数，如3列 |
| 4 | 确认屏幕总数 | 系统根据行列数生成布局网格 |
| 5 | 点击「保存」 | 保存显示屏物理布局 |

提示：行数和列数必须与物理大屏的实际拼接方式完全一致，否则场景画面会错位。

### 6.2端口映射

将拼控主机的物理输出端口与大屏的屏幕位置进行自动顺序对应。

对应规则：

系统按照物理布局的行列顺序，自动依次对应拼控主机的HDMI输出端口：

| 位置 | 自动对应端口 | 物理屏幕 |
| --- | --- | --- |
| 第1行第1列 | HDMI-Out-1 | 左上角屏幕 |
| 第1行第2列 | HDMI-Out-2 | 中上屏幕 |
| 第1行第3列 | HDMI-Out-3 | 右上角屏幕 |
| 第2行第1列 | HDMI-Out-4 | 左下角屏幕 |
| 第2行第2列 | HDMI-Out-5 | 中下屏幕 |
| 第2行第3列 | HDMI-Out-6 | 右下角屏幕 |

对应规则：按行优先顺序，从左到右、从上到下依次对应HDMI-Out-1、HDMI-Out-2、HDMI-Out-3……

提示：物理接线时请按照上述顺序连接HDMI线缆，确保第1行第1列接HDMI-Out-1，第1行第2列接HDMI-Out-2，以此类推。接线完成后可点击「测试输出」验证每块屏幕位置是否正确。

### 6.3错接校正

当物理端口接错时，无需重新插拔HDMI线，可通过软件校正。

| 场景 | 操作 | 效果 |
| --- | --- | --- |
| 两屏位置互换 | 在端口映射中交换两个格子的端口选择 | 画面自动交换到正确位置 |

提示：错接校正只改变信号输出到哪个屏幕，不改变场景内容。校正后所有场景自动按新映射显示。

## 七、场景预设（核心）

场景是运行中心显控系统的核心配置单元。一个场景=窗口布局+信号源绑定+轮巡策略。前端展示顺序和可见性由场景列表统一配置。

场景管理操作均在一级目录（场景卡片列表）完成，点击场景卡片进入二级目录进行场景内容编辑。

### 7.1场景管理（一级目录操作）

在「场景预设」→「场景列表」一级目录中，可进行以下场景管理操作：

| 操作 | 说明 | 入口 |
| --- | --- | --- |
| 新建空白场景 | 创建新场景，进入场景搭建向导 | 场景卡片列表中点击「新建场景」按钮 |
| 编辑场景 | 修改场景内容（窗口布局、信号源绑定、轮巡策略等） | 场景卡片上点击「编辑」→进入二级目录 |
| 删除场景 | 删除场景，前端不再显示 | 场景卡片上点击「删除」 |
| 另存为新场景 | 基于现有场景快速创建新场景 | 场景卡片上点击「复制」 |
| 调整顺序 | 调整场景卡片在前端的排列顺序 | 拖拽场景卡片上下/左右移动 |

提示：编辑场景并保存后，前端操作员立即看到更新后的场景内容。后台场景与大屏实时同步，若该场景正在大屏显示，保存后大屏会实时更新。删除场景前需确认是否影响前端操作。

新建场景步骤：

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 点击「新建场景」 | 弹出场景配置向导 |
| 2 | 填写场景名称 | 如「智能巡课」「接待欢迎」 |
| 3 | 填写场景描述 | 说明场景用途，便于前端操作员理解该场景内容 |
| 4 | 点击「下一步」 | 进入场景搭建（二级目录） |

提示：场景描述建议填写清晰易懂的内容，如「巡课模式-3×3九画面显示各教室前视摄像头」，方便前端操作员快速识别场景用途。

### 7.2前端展示顺序配置

控制前端操作界面中场景Tab的显示顺序和可见性。

界面结构：

一级目录：场景卡片列表

- 场景A、场景B、场景C（场景卡片）
- 支持拖拽调整场景卡片顺序
- 支持开关控制场景是否在前端显示
- 点击场景卡片进入二级目录
二级目录：场景内容编辑

- 场景名称、场景描述
- 具体场景内容编辑区域（场景布局/窗口绑定信号源等）
场景卡片信息：

| 信息项 | 说明 |
| --- | --- |
| 场景名称 | 场景的标识名称，如「智能巡课」 |
| 场景描述 | 说明场景用途，便于前端操作员理解该场景内容 |
| 显示开关 | 控制该场景是否在前端操作界面显示 |

配置操作：

| 配置项 | 作用 | 操作 |
| --- | --- | --- |
| 排序 | 调整场景卡片在前端Tab栏的排列顺序 | 拖拽场景卡片上下/左右移动 |
| 可见性 | 控制场景是否在前端显示 | 开关控制，关闭后前端操作员看不到该场景 |
| 默认场景 | 设置系统启动后默认显示的场景 | 勾选「设为默认」，系统启动后自动调用该场景 |
| 进入编辑 | 编辑场景具体内容 | 点击场景卡片进入二级目录，编辑窗口布局、信号源绑定、轮巡策略等 |

提示：建议将最常用的场景排在最前面，并设为默认场景。场景描述建议填写清晰易懂的内容，方便前端操作员快速识别。

### 7.3场景搭建（二级目录编辑）

点击场景卡片的「编辑」或新建场景后进入二级目录，在空白场景画布上进行场景搭建。

操作项目：

| 操作 | 说明 |
| --- | --- |
| 开窗 | 在画布上创建新窗口，支持普通窗口和漫游窗口 |
| 窗口漫游 | 拖拽窗口自由移动位置，漫游窗口支持自动吸附边缘 |
| 窗口缩放 | 拖拽窗口边框调整大小 |
| 窗口分割 | 4分割/9分割/16分割，等比例分割模式 |
| 信号源绑定 | 将信号源拖拽到目标窗口进行绑定（详见7.4） |
| 保存场景 | 保存为当前场景（覆盖）或另存为新场景 |

等比例分割模式：

| 布局模式 | 窗口数 | 适用场景 |
| --- | --- | --- |
| 4分割 | 4 | 2×2四画面，如4教室对比 |
| 9分割 | 9 | 3×3九画面，如楼层巡课 |
| 16分割 | 16 | 4×4十六画面，如大规模监控 |

注意：等比例分割模式下，所有窗口大小相同、整齐排列，不支持漫游和叠加。漫游窗口不支持等比例分割，仅支持自由拖拽布局。

自定义漫游模式：

- 在画布上拖拽创建窗口
- 拖拽窗口边框调整大小
- 拖拽窗口位置实现漫游，支持自动吸附边缘
- 支持窗口跨屏（窗口可跨越多个物理屏幕）
注意：窗口布局一旦保存到场景，前端操作员只能切换场景，不能修改窗口大小位置。如需调整，必须回到管理员界面编辑场景。

### 7.4信号源绑定

将场景中的每个窗口与具体信号源绑定。

绑定方式：

| 方式 | 操作 | 说明 |
| --- | --- | --- |
| 拖拽绑定 | 在左侧结构树中长按信号源，直接拖拽到目标窗口 | 最快捷的绑定方式，所见即所得 |

信号源预览：

- 绑定前支持预览信号源画面，确认内容正确后再绑定
- 在信号源选择面板中，悬停或点击信号源缩略图可查看实时预览
绑定步骤：

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 在左侧结构树中找到目标信号源 | 支持搜索快速定位 |
| 2 | 长按该信号源并拖拽 | 信号源支持预览 |
| 3 | 将信号源拖放到目标窗口，释放完成绑定 | 窗口上显示信号源号 |
| 4 | 检查窗口画面 | 确认窗口已显示正确的信号源预览 |
| 5 | 重复步骤1-4，为所有窗口绑定信号源 | 未绑定的窗口显示黑屏或默认画面 |
| 6 | 点击「保存场景」或「另存为新场景」 | 保存场景配置，并同步到大屏 |

提示：一个信号源可同时绑定到多个场景的不同窗口。修改信号源配置（如RTSP地址）会影响所有绑定该信号源的场景。

### 7.5轮巡策略配置

为场景配置轮巡功能，实现自动切换画面。轮巡策略内嵌在场景中：后台负责配置轮巡规则，前端只负责在支持轮巡的场景中开启、暂停和选择筛选范围。

#### 7.5.1自定义轮巡

最灵活的轮巡模式，选择1个/多个窗口，选择多个信号源，设置轮巡间隔，选择的窗口联动轮巡选择的多个信号源。

操作步骤：

- 在场景编辑界面，点击「轮巡配置」→「自定义轮巡」
- 选择目标窗口（该窗口将用于轮巡显示）
- 从信号源列表中选择多个信号源加入轮巡列表
- 设置轮巡间隔
- 点击「保存」
#### 7.5.2单机位轮巡

以教室为单元，选择机位（单选）进行轮巡。

适用场景：需要轮巡查看各教室的同一类画面，如只看每间教室的老师画面、或只看学生画面。

机位选择：

以教室为单元，从以下机位中选择1个：

- 老师机位
- 学生机位
- 课件机位
配置项：

| 配置项 | 说明 |
| --- | --- |
| 单元分组 | 按教室分组（如教室101、教室102、教室103...） |
| 选择机位 | 从老师机位/学生机位/课件机位中选择1个 |
| 轮巡窗口 | 指定场景中窗口轮巡 |
| 切换间隔 | 轮巡切换拼接屏整体页面时间间隔 |
| 课表联动 | 可联动课表数据，只轮巡当前有课教室 |

操作步骤：

- 在场景编辑界面，点击「轮巡配置」→「单机位轮巡」
- 选择单元分组方式（按结构树节点自动按教室分组）
- 选择1个机位（如只选「老师机位」）
- 选择1个窗口用于显示轮巡画面
- 选择是否启用课表联动筛选（如只轮巡有课教室）
- 设置切换间隔
- 点击「保存」
#### 7.5.3多机位绑定轮巡

以教室/房间为单元，每个单元选择多个机位（多选，2个或3个），多个窗口成组绑定，轮巡时一组窗口同时切换。

适用场景：需要同时查看每间教室的多个视角，如老师画面+学生画面+课件画面同时展示。

机位选择：

以教室为单元，从以下机位中选择多个（2个或3个）：

- 老师机位
- 学生机位
- 课件机位
窗口绑定规则：

- 选择N个机位，就需要绑定N个窗口
- 每个机位对应一个窗口，一一对应绑定

| 机位选择数 | 需要窗口数 | 绑定关系 |
| --- | --- | --- |
| 2个机位 | 2个窗口 | 窗口1=老师机位，窗口2=学生机位 |
| 3个机位 | 3个窗口 | 窗口1=老师机位，窗口2=学生机位，窗口3=课件机位 |

配置项：

| 配置项 | 说明 |
| --- | --- |
| 单元分组 | 按教室分组（如教室101、教室102、教室103...） |
| 选择机位 | 从老师机位/学生机位/课件机位中选择2个或3个 |
| 窗口绑定 | 将选择的机位绑定到对应数量的窗口（机位数=窗口数） |
| 切换间隔 | 轮巡切换拼接屏页面时间间隔 |
| 课表联动 | 可联动课表数据，动态更新轮巡教室范围 |

操作步骤：

- 在场景编辑界面，点击「轮巡配置」→「多机位绑定轮巡」
- 选择多个机位（如选择「老师机位」和「学生机位」）
- 绑定对应数量窗口组（按上述选择的机位，则两个窗口为一组绑定）
- 选择是否启用课表联动筛选
- 设置切换间隔
- 点击「保存」
注意：多机位绑定轮巡要求选择的机位数与绑定的窗口数严格一致。如选择2个机位，必须绑定2个窗口；选择3个机位，必须绑定3个窗口。

## 八、系统设置

### 8.1网络与IP配置（安卓端）

本模块用于配置安卓触控终端（Pad）的网络IP，确保Pad与解码拼控主机在同一网段且网络互通。

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| IP获取方式 | DHCP自动获取/静态固定IP | DHCP |
| 静态IP地址 | Pad的固定IP地址 | 空（DHCP时无效） |
| 子网掩码 | 网络子网掩码 | 空 |
| 网关 | 网络网关地址 | 空 |
| DNS | 域名解析服务器 | 空 |

配置要求：

- 触控终端的IP地址必须与解码拼控主机的IP地址在同一网段
- 配置完成后，需确保Pad能ping通拼控主机的IP地址
- 建议使用静态IP，避免DHCP分配变化导致连接中断
提示：修改IP后若无法连接拼控主机，请检查两者是否在同一网段，网络是否互通。

### 8.2密码管理

| 配置项 | 说明 | 操作 |
| --- | --- | --- |
| 管理员密码 | 6位纯数字密码，进入后端配置界面的凭证 | 输入旧密码+新密码+确认新密码 |
| 操作员免密 | 是否允许操作员免密码进入前端 | 开关控制，默认开启 |

提示：修改密码后请牢记，忘记密码需恢复出厂设置才能重置。

### 8.3版本升级

支持在线升级和U盘离线升级两种方式。

| 方式 | 步骤 | 适用场景 |
| --- | --- | --- |
| 在线升级 | 系统设置→版本升级→检查更新→下载并安装 | 设备可连接互联网 |
| U盘升级 | 将升级包拷贝到U盘→插入拼控主机USB口→系统设置→版本升级→选择U盘升级 | 设备无法联网或内网环境 |

提示：升级过程中请勿断电，升级完成后系统会自动重启。建议升级前导出配置备份。

### 8.4存储与日志

| 配置项 | 说明 | 操作 |
| --- | --- | --- |
| 存储空间查看 | 查看系统存储使用情况（视频缓存、日志、配置等） | 实时显示，无需操作 |
| 日志导出 | 将系统运行日志导出到U盘 | 插入U盘→点击「导出日志」→选择导出范围（最近7天/30天/全部） |
| 格式化存储 | 清空所有缓存数据和日志（保留配置） | 点击「格式化」→二次确认 |

提示：日志导出对排查问题非常重要。遇到异常时，请导出全部日志并联系技术支持。

### 8.5初始化恢复（恢复出厂）

将系统恢复到出厂默认状态，所有配置数据将被清空。

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 进入「系统设置」→「初始化恢复」 | - |
| 2 | 阅读警告提示 | 明确告知该操作会清空所有信号源、场景、屏幕配置 |
| 3 | 输入管理员密码确认 | 防止误操作 |
| 4 | 点击「确认恢复出厂」 | 系统重置并重启 |

警告：恢复出厂后，所有配置数据丢失，需重新配置信号源、显示屏、场景。操作前请务必确认已备份或不再需要现有配置。

## 九、配置示例

### 9.1示例：智能巡课

场景需求：9个屏幕单元，每个屏幕单元9分割布局，显示81间教室的摄像头画面，联动课表轮巡查看所有上课教室画面。

配置步骤：

- 准备工作：通过ipconfig设置拼控主机和编码器的固定IP
- 主控与外接网口配置：在App中填写拼控主机主控IP，按需配置外接网口IP
- 信号源管理：添加学校所有教室的摄像头RTSP地址并按照规则命名
- 平台同步：同步课表数据，用于动态筛选当前有课教室
- 显示屏配置：设置3×3布局，9块屏幕自动对应9个HDMI输出端口
- 场景预设：
- 新建场景「巡课场景」
- 每个屏幕单元开窗选择9分割布局
- 配置单机位轮巡：选择老师机位，启用课表联动，动态轮巡当前有课教室，轮巡间隔设置为15s
- 前端展示：设为可见，排序第一位
### 9.2示例：多机位绑定轮巡

场景需求：每间教室3个摄像头（老师、学生、课件），轮巡时每间教室同时展示3个视角。

配置步骤：

- 准备工作：通过ipconfig设置拼控主机和编码器的固定IP
- 主控与外接网口配置：在软件中填写拼控主机主控IP，按需配置外接网口IP
- 信号源管理：添加所有教室的摄像头，按规则填写信号源名称，自动分类生成结构树
- 显示屏配置：设置3×3布局，每行3个窗口对应1间教室的3个视角
- 场景预设：
- 新建场景「多机位巡课」
- 选择9分割布局
- 配置多机位绑定轮巡：
- 每组3个机位（老师、学生、课件）
- 绑定每一行的3个窗口
- 按教室分组，每次切换展示一间教室的3个视角
- 切换间隔15秒
- 前端展示：设为可见，排序第三位
